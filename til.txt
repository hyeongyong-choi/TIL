1. 앞으로 git 프로젝트로 만들고 싶은 것
- 팀원들과 토이프로젝트 협업하여 만들기
- 유튜브 / 네이버와같은 사이트 클론코딩
- 가상의 사이트 만들어보기

2. 1주차 주요 개념 키워드 적어보기
- 버전 관리 (깃프로젝트로 관리시에 용이함)
- git 초기화 (깃프로젝트로 변경하는 것)
- commit (누가 , 언제 , 어떻게와 같은 프로젝트 상황 파악)
- add or staging (commit에 반영할 파일을 선택)
- repo (깃으로 관리되는 프로젝트를 뜻하며 내 컴퓨터에 저장된 리포지토리는 로컬repo , 깃허브처럼 다른곳에서 접속할 수 있는 공간에 저장되어 있는 것을 원격repo라고 함)
- tracking (로컬repo와 원격repo를 연결)
- push (로컬repo의 commit들을 원격repo에 반영하는 것)
- pull (원격repo의 commit들을 로컬repo에 반영하는 것)
- clone (로컷repo에 commit된 정보가 없을때, 원격 repo를 내 컴퓨터로 가져와서 초기 repo세팅을 하는 것 복제라고 보면 됨)
